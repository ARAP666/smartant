# Registro de decisiones

| Fecha | Decisión | Motivo |
| --- | --- | --- |
| 2026-06-19 | El MVP será móvil con React Native y Expo Go. | Permite validar el flujo antes de preparar EAS. |
| 2026-06-19 | La API será Express con TypeScript. | Requisito explícito y alcance backend moderado. |
| 2026-06-19 | PostgreSQL será local en desarrollo y Railway alojará el backend posteriormente. | Requisito de operación y despliegue. |
| 2026-06-19 | Los gastos entrarán manualmente, por fotografía o por CSV/XLSX. | Son los tres canales requeridos. |
| 2026-06-19 | Toda captura o importación requerirá revisión antes de persistir. | Evita que datos detectados incorrectamente alteren dinero real. |
| 2026-06-19 | Las alertas internas forman parte del MVP; push queda fuera hasta EAS. | Expo Go permite validar primero el valor sin infraestructura adicional. |
| 2026-06-19 | Se usará Lucide; no se usarán iconos Unicode. | Requisito visual explícito. |
| 2026-06-19 | La marca inicial será únicamente tipográfica. | No se requiere logo gráfico. |
| 2026-06-19 | Splash e identidad interna reutilizan la misma marca tipográfica. | Evita identidades duplicadas y mantiene consistencia visual. |
| 2026-06-20 | La base PostgreSQL local se llama `sentDB`. | Nombre definido por el usuario para desarrollo local. |
