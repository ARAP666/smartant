## Dificultades Y Ajustes

- No se intento desplegar a Railway porque credenciales, dominio y valores finales se entregaran hasta el handoff EAS/APK/produccion.

- El contenedor usa `tsx src/server.ts` porque `@smart-ant/finance` aun exporta TypeScript fuente.
- Existen advisories moderados transitivos; las correcciones automaticas implican breaking upgrades y se difieren.

## Deuda Y Riesgos

- Produccion real sigue pendiente hasta recibir credenciales Railway/EAS y valores finales.
- El rate limit en memoria debe reforzarse con edge/proveedor si la API queda publica.

## Aprendizajes

- Preparar scripts, docs y gates antes de credenciales reduce friccion del handoff final.
- No conviene bloquear tareas locales por secretos o infraestructura que se entregaran al ultimo momento.
- Las notas temporales deben estar visibles en produccion-readiness y retros para que no se pierdan.
- Operabilidad minima sin dependencias externas fue suficiente para el estado actual.
